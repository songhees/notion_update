명령어

exit : session 종료 ( = Ctrl+C = /clear )

/btw : 클로드가 한창 코딩이나 리팩토링 작업을 수행 중일 때, 작업 흐름을 방해하지 않으면서 별도의 사이드 대화를 나눌 수 있는 기능

/model : 사용하려는 모델 설정

Claude Code 란

Claude LLM 을 감싸서 코딩할 수 있는 환경을 제공하는 코딩 도구

에이전트 기반 코딩 도구 → terminal agent loop, context 관리 창, 도구 및 구성 권한 등

Claude와 달리 직접적으로 터미널과 파일, IDE 에 접속해 직접 작업을 처리할 수 있다.

프롬프트 → 컨텍스트 수집 → 작업 수행 → 결과 검증

Claude code가 사용할 수 있는 도구 (ex terminal, bash, file …)

  1. 코드베이스를 읽기
  1. 파일을 편집
  1. 명령 실행
  1. 개발 도구와 통합

하는 에이전트 코딩 도구 / 코딩 어시스턴트

terminal, IDE, 데스크톱 앱 및 브라우저에서 사용가능

작동 방식

사용자의 명령(프롬프트)

→ 프롬프트를 충족시키기 위한 context 수집 : claude code 가 상호작용 할 수 있는 텍스트 또는 도구 호출을 하는 모델과 상호 작용함으로써 이뤄진다.

→ 조치를 취하기 : 파일 편집, 명령어 실행

→ 결과 검증 : 해당 결과가 프롬프트에서 의도한 바를 달성했는지 여부를 판단.

달성 ⇒ 다음 프롬프트 대기

미달성 ⇒ 검증이 가능해질 때 까지 loop 를 돌면서 같은 과정을 반복한다.

Context

claude의 작업 memory

대화 내용, 파일 내용, 명령 출력 등을 저장하고 나중에 다시 볼 수 있다.

→ 저장할 수 있는 양이 정해져 있기 때문에 이를 최적화하는 것이 중요하다.

→ 용량의 한계(95%)에 도달하면 자동 압축을 하나 품질이 상당히 저하

중요한 세부 정보를 요약하고 불필요한 도구 호출 결과를 제거하여 공간을 확보

→ 단, 이전 대화 내용이 손실될 수 있다.

적용 방안

  • Yes, auto-accept edits
    수정 사항을 자동으로 승인하고 코드에 즉시 반영합니다. 하나하나 확인할 필요 없이 AI가 제안한 대로 바로 업데이트하고 싶을 때 선택합니다.
  • Yes, manually approve edits
    수정 사항을 수동으로 확인한 뒤 승인합니다. 코드가 어떻게 변하는지 직접 눈으로 확인(Diff 체크)하고 반영 여부를 결정하고 싶을 때 사용합니다.
  • No, refine with Ultraplan on Claude Code on the web
    현재 제안이 마음에 들지 않을 때 선택합니다. 웹 버전의 Claude에서 더 성능이 좋은 모델(Ultraplan 등)을 사용해 수정안을 다시 다듬고 싶다는 의미입니다.
  • Tell Claude what to change
    Claude에게 구체적으로 어떤 부분을 다시 고쳐야 할지 지시를 내립니다. 특정 로직이 잘못되었거나 추가 요구사항이 있을 때 사용합니다.

filter-test-output.sh

클로드에서 표출하게되는 에러 코드를 줄여줘서 발생하는 토큰 수를 줄이는 방법

#!/bin/bash
input=$(cat)
cmd=$(echo "$input" | jq -r '.tool_input.command')

# 테스트 명령어 감지 (Javascript, Python, Go, Spring boot)
if [[ "$cmd" =~ ^(npm test|pytest|go test|./mvnw test|./gradlew test|mvn test|gradle test) ]]; then

  # Spring Boot (Maven/Gradle) 전용 필터
  if [[ "$cmd" =~ ^(./mvnw|mvn|./gradlew|gradle) ]]; then
    filtered_cmd="$cmd 2>&1 | grep -A 10 -E '(FAILED|ERROR|Tests run.*Failures|BUILD FAILURE|Caused by:|at .+\(.+\.java)' | head -150"

  # 기존 필터 (npm/pytest/go)
  else
    filtered_cmd="$cmd 2>&1 | grep -A 5 -E '(FAIL|ERROR|error:)' | head -100"
  fi

  echo "{\"hookSpecificOutput\":{\"hookEventName\":\"PreToolUse\",\"permissionDecision\":\"allow\",\"updatedInput\":{\"command\":\"$filtered_cmd\"}}}"
else
  echo "{}"
fi
{
  "hookSpecificOutput": {           // 1. 훅 답장 시작합니다!
    "hookEventName": "PreToolUse",  // 2. 실행 직전 상황에 대한 보고고요,
    "permissionDecision": "allow",  // 3. 실행은 허가(allow)하겠습니다.
    "updatedInput": {               // 4. 단, 입력값은 이걸로 바꿔주세요:
      "command": "$filtered_cmd"    // 5. 명령어(command) 자리에 이 필터링된 내용을 넣으세요!
    }
  }
}

즉 클로드가 실행하려고한 실제 명령어인 "./gradlew test”를 "./gradlew test 2>&1 | grep -A 10 -E '(FAILED|ERROR|...)' | head -150”
